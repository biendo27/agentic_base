```text

├── app/          # Shared app-shell translations
└── home/         # Starter feature translations

lib/
├── app/          # Bootstrap, flavors, generated i18n
├── core/         # DI, network, theme, router, error handling
```

## CI/CD Contract

This scaffold persists `ci_provider: {{ci_provider}}` in `.info/agentic.yaml` and emits exactly one CI provider contract:

- `github`: `.github/workflows/*.yml`, `gh auth login`, and `agentic_base deploy <env>` triggers `cd-<env>.yml`
- `gitlab`: root `.gitlab-ci.yml` plus `.gitlab/ci/*.yml`, `glab auth login`, and `agentic_base deploy <env>` targets `deploy_<env>`

GitLab native validation is macOS-only by contract:

- register a project runner with `tags: [macos]`
- use a shell executor on a macOS machine
- install Xcode on that runner
- do not treat Linux runners as a replacement for iOS/native validation
