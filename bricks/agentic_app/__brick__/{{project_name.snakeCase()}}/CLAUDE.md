# CLAUDE.md

## Project
Flutter app generated by agentic_base with Cubit state management.

## Runtime Contract
- App shell lives under `lib/app/**` and `lib/main*.dart`
- Localization source lives under `assets/i18n/**`
- Generated localization code lives under `lib/app/i18n/translations.g.dart`
- Runtime config comes from `env/*.env.example` via `String.fromEnvironment`

## Commands
- `flutter pub get` — install dependencies
- `./tools/gen.sh` — clean i18n, run build_runner, materialize Slang output
- `flutter run --flavor dev -t lib/main_dev.dart --dart-define-from-file=env/dev.env.example` — run dev flavor
- `flutter test` — run tests
- `dart analyze` — lint check

## Conventions
- snake_case file names
- package imports only
- keep files under 200 LOC when practical
- add feature translations under `assets/i18n/<feature>/<feature>_<locale>.i18n.yaml`
- keep `.info/agentic.yaml` as the single source of truth for `ci_provider`

## Boundaries
- never edit `*.g.dart`, `*.freezed.dart`, `*.gr.dart`, `*.config.dart`
- never commit secrets or non-example env files
- never commit `.idea` files outside `.idea/runConfigurations`

## CI/CD
- This scaffold is generated with `ci_provider: {{ci_provider}}`
- GitHub projects use `.github/workflows/*.yml` and require authenticated `gh`
- GitLab projects use root `.gitlab-ci.yml` plus `.gitlab/ci/*.yml` and require authenticated `glab`
- GitLab native validation depends on a macOS shell runner with Xcode and `tags: [macos]`
